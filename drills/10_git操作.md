## 準備

gitsigns は git リポジトリ内でのみ動作します。このリポジトリはすでに git 管理されています。

1. 練習用ファイルをコピーして変更を加える準備をします：

```
cp practice/08_LSP_practice.py practice/10_git操作_practice.py
```

2. Neovim で開いてください：

```
nvim practice/10_git操作_practice.py
```

---

## 練習問題

### Q1. hunk（変更箇所）を確認する

1. ファイルに何行か変更を加えてください（例: `# コメントを追加` を数か所に挿入、または行を削除）
2. サインカラム（行番号の左）に `+` / `-` / `~` が表示されていることを確認してください
3. `]c` を押してください（次の変更 hunk へジャンプ）
4. `[c` を押してください（前の変更 hunk へ戻る）

> **確認**: gitsigns が動いていない場合は `:checkhealth gitsigns` を実行してください。

---

### Q2. `<leader>hp` で hunk のプレビューを表示する

1. 変更した行の hunk 内にカーソルを置いてください
2. `<leader>hp` を押してください（変更前後の diff がポップアップ表示される）
3. `q` または `Esc` でポップアップを閉じてください

> **目標**: 変更内容を確認するために `:!git diff` を打たず `<leader>hp` で確認する。

---

### Q3. `<leader>hd` で diff を表示する

1. hunk 内にカーソルを置いてください
2. `<leader>hd` を押してください（diff ビューが開く）
3. 変更前後の内容を確認してください
4. `:q` で diff ビューを閉じてください

---

### Q4. `<leader>hs` で hunk をステージする

1. ステージしたい hunk にカーソルを置いてください
2. `<leader>hs` を押してください
3. サインカラムの表示が変わったことを確認してください（ステージ済みを示す色に変わる）

> **ヒント**: hunk 単位のステージは `git add -p` より速く、特定の変更だけをコミットしたいときに使います。

---

### Q5. `<leader>hr` で hunk を元に戻す

1. 変更した hunk にカーソルを置いてください
2. `<leader>hr` を押してください（その hunk の変更が取り消される）
3. サインカラムから表示が消えたことを確認してください

> **注意**: `<leader>hr` は git の変更を取り消すので `u` ではアンドゥできません。

---

### Q6. `<leader>hb` で行の git blame を表示する

1. 任意の行にカーソルを置いてください
2. `<leader>hb` を押してください（その行の最終コミット情報がインラインで表示される）
3. 誰がいつ・どのコミットで変更したかを確認してください

---

### Q7. 総合チャレンジ

1. `practice/10_git操作_practice.py` に複数の変更を加えてください（行追加・削除・修正を混在させる）
2. `]c` / `[c` で変更箇所を巡回してください
3. `<leader>hp` で各 hunk の変更内容を確認してください
4. 意図した変更だけを `<leader>hs` でステージしてください
5. 意図しない変更は `<leader>hr` で元に戻してください
6. `:!git status` でステージ状況を確認してください
