# Neovim エンジニア実践リファレンス（kickstart.nvim）

> リーダーキー = `Space`　／　kickstart.nvim デフォルト設定

---

## 1. 絶対に覚える移動

```
h j k l          ← ↓ ↑ →（基本は使わず下記を使う）
w / b            単語の先頭へ前進 / 後退
e                単語の末尾へ
0 / ^            行頭 / 非空白の行頭
$                行末
gg / G           ファイル先頭 / 末尾
Ctrl-d / Ctrl-u  半画面スクロール下 / 上（最重要）
{ / }            段落（空行）を前後にジャンプ
%                対応する括弧へジャンプ
f{c} / F{c}      行内で文字 c へ前進 / 後退
```

---

## 2. モード切り替え

```
i / I    カーソル前 / 行頭から挿入
a / A    カーソル後 / 行末から挿入
o / O    下に新行を追加して挿入 / 上に新行を追加して挿入
v / V    ビジュアル / 行ビジュアル
Ctrl-v   矩形ビジュアル（複数行を列で選択）
Esc      ノーマルモードに戻る
```

---

## 3. 編集の基本パターン（動詞 + 名詞）

### オペレータ（動詞）

| キー | 意味 |
|------|------|
| `d`  | 削除（Delete） |
| `c`  | 変更（Change）= 削除して挿入 |
| `y`  | ヤンク（Yank）= コピー |

### モーション / テキストオブジェクト（名詞）

| キー | 範囲 |
|------|------|
| `w`  | 単語 |
| `$`  | 行末まで |
| `iw` | 単語の内側（空白を含まない） |
| `aw` | 単語 + 前後空白 |
| `i"` | `"..."` の内側 |
| `a"` | `"..."` 全体 |
| `i(` | `(...)` の内側 |
| `it` | HTML タグの内側 |
| `ip` | 段落全体 |

### 組み合わせ例（毎日使う）

```
dw    単語を削除
dd    行を削除
D     行末まで削除
cw    単語を変更（消して挿入モードへ）
cc    行全体を変更
ci"   ダブルクォート内を変更（設定ファイル編集に最頻出）
ci(   括弧内を変更
yy    行をコピー
p / P カーソルの後 / 前にペースト
```

## 4. 繰り返しの3種

```
.        直前の変更を繰り返す（最重要。これを軸に考える）
u        アンドゥ
Ctrl-r   リドゥ

# マクロ
qa       レジスタ a にマクロ記録開始
（操作）
q        記録終了
@a       マクロ再生
10@a     10回繰り返す
```

## 5. 検索・置換

### 検索

```
/pattern     前方検索
?pattern     後方検索
n / N        次 / 前のマッチへ
*            カーソル下の単語を検索（最速）
:noh         ハイライトを消す
```

### cgn パターン（単語の一括変更に最速）

```
*       カーソル下の単語を検索
cgn     次のマッチを変更して挿入モードへ
Esc
.       次のマッチにも同じ変更（スキップしたい時は n）
```

### 置換

```
:%s/old/new/g     ファイル全体を置換
:%s/old/new/gc    確認しながら（y=置換 / n=スキップ / a=全部 / q=終了）
:'<,'>s/old/new/g ビジュアル選択範囲を置換
```

## 6. ファイル操作・保存・終了

```
:w        保存
:wa       全バッファを保存
:q        閉じる（変更なし）
:wq / ZZ  保存して閉じる
:q!  / ZQ 保存せずに閉じる
:qa       全ウィンドウを閉じる
:e {file} ファイルを開く
:e!       ファイルを再読み込み（外部変更を取り込む）
```

## 7. telescope（kickstart のメイン機能）

```
<leader>sf   ファイルを検索して開く（最頻出）
<leader>sg   プロジェクト全体を grep（ripgrep 必須）
<leader>sw   カーソル下の単語でプロジェクト grep
<leader>/    現在バッファ内をファジー検索
<leader>sb   開いているバッファ一覧
<leader>sr   最近開いたファイル
<leader>sh   help を検索
<leader>sd   エラー・警告一覧
<leader>sk   キーマップを検索（忘れた時に）
```

telescope が開いている中での操作：
```
Ctrl-j / Ctrl-k  候補を上下移動
Enter            開く
Ctrl-v           垂直スプリットで開く
Ctrl-x           水平スプリットで開く
Ctrl-t           タブで開く
Esc              閉じる
```

## 8. LSP（コードを書く核心）

### ジャンプ系

```
gd          定義へジャンプ（最頻出。関数の定義を見に行く）
gD          宣言へジャンプ
gr          参照一覧（どこから使われているか）
gI          実装へジャンプ（interface → 実装クラス）
Ctrl-o      ジャンプ前の位置に戻る（gd の後で必ず使う）
```

### コード操作

```
K               ホバードキュメント（型・説明を表示）
<leader>rn      シンボルをリネーム（変数名・関数名の一括変更）
<leader>ca      コードアクション（import 追加・自動修正など）
<leader>ds      現在ファイルのシンボル一覧（telescope）
```

### 診断（エラー・警告）

```
]d / [d         次 / 前の診断へ移動
]e / [e         次 / 前のエラーへ移動
<leader>e       カーソル位置の診断を浮かびウィンドウで表示
<leader>q       全診断を quickfix リストに開く
```

## 9. ウィンドウ・バッファ

### ウィンドウ分割

```
:vsp / :sp      垂直 / 水平分割
Ctrl-w v        垂直分割（ショートカット）
Ctrl-h/j/k/l    ウィンドウ間を移動（kickstart でマップ済み）
Ctrl-w =        全ウィンドウを均等サイズに
Ctrl-w q        現在のウィンドウを閉じる
```

### バッファ管理

```
:ls             開いているバッファ一覧
:b{n}           バッファ番号 n に切り替え
:b <Tab>        ファイル名補完で切り替え
Ctrl-^          直前のバッファに切り替え（最頻出）
:bd             現在バッファを削除（閉じる）
:%bd|e#         現在ファイル以外を全部閉じる
```

## 10. git 操作（gitsigns / kickstart 同梱）

```
]c / [c         次 / 前の変更 hunk へ移動
<leader>hs      hunk をステージ
<leader>hr      hunk を元に戻す
<leader>hp      hunk のプレビューを表示
<leader>hb      行の git blame を表示
<leader>hd      diff を表示（変更前後を確認）
```

## 11. その他の毎日使うコマンド

### コメント（mini.nvim 同梱）

```
gcc     行をコメントトグル
gc      + モーション（例: gcip で段落全体）
gc      ビジュアル選択範囲をコメントトグル
```

### インデント

```
>>  /  <<   行を右 / 左にインデント
> / <       ビジュアル選択をインデント（Vip> で段落ごと）
=G          カーソルからファイル末尾まで自動インデント
```

### quickfix（LSP の診断・grep の結果を巡る）

```
:copen      quickfix リストを開く
:cn / :cp   次 / 前の項目へ移動
:cclose     quickfix を閉じる
```

### 便利なその他

```
Ctrl-a / Ctrl-x   カーソル下の数値を +1 / -1
~                 大文字 ↔ 小文字を切り替え
J                 次の行を現在行に結合
ga                カーソル下の文字コードを表示
Ctrl-o（挿入中）  挿入モードのまま1コマンドだけ実行
:terminal         組み込みターミナルを開く
```

## 12. 典型的な作業フロー

### 関数を修正する

```
1. <leader>sf でファイルを開く
2. /funcname で関数を検索
3. gd で定義へジャンプ
4. K でシグネチャを確認
5. ci( で引数を変更
6. <leader>rn で呼び出し側も一括リネーム
7. Ctrl-o で元の位置に戻る
```

### プロジェクト内の文字列を一括変更する

```
1. <leader>sg で grep して対象を確認
2. <leader>sw でカーソル下の単語を grep（より速い）
3. cgn で1つ目を変更 → . で次々適用
   または :%s/old/new/gc で確認しながら置換
```

### エラーを修正する

```
1. ]e でエラーへジャンプ
2. <leader>e でエラーメッセージを確認
3. <leader>ca でコードアクション（自動修正が使えることも）
4. [e / ]e で前後のエラーへ移動しながら修正
```

### ファイルを並べて比較・編集する

```
1. :vsp でウィンドウを垂直分割
2. <leader>sf でもう一方のファイルを開く
3. Ctrl-h / Ctrl-l でウィンドウを行き来
4. Ctrl-w = でサイズを揃える
```

## 13. 困ったときのコマンド

```
:checkhealth        nvim・プラグインの健康診断
:LspInfo            接続中の LSP サーバーを確認
:Lazy               プラグイン管理（U で全更新）
:Mason              LSP・フォーマッタの管理
:help {topic}       組み込みヘルプ（例: :help gd）
<leader>sk          キーマップを検索（忘れた時）
:messages           直前のメッセージ一覧
:verbose map {key}  そのキーに何が割り当たっているか確認
```

## 14. init.lua を最初に変えるべき設定

```lua
-- ~/.config/nvim/init.lua の options セクションに追加

vim.opt.relativenumber = true  -- 相対行番号（ジャンプが楽になる）
vim.opt.scrolloff = 8          -- 上下に8行の余白を常に確保
vim.opt.wrap = false           -- 長い行を折り返さない
vim.opt.tabstop = 2            -- タブ幅 2（言語に合わせて調整）
vim.opt.shiftwidth = 2
vim.opt.colorcolumn = '100'    -- 100桁ガイド線
vim.opt.undofile = true        -- アンドゥ履歴をファイルに永続化
```
